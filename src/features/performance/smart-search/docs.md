## Ideas de presentación de alto impacto

- “Panel técnico” en vivo: cards con Debounce: 300ms, API calls, Cache hit ratio, Tiempo de filtro (ms)
- Botones demo (“Juan” → “Pedro” → “Juan”) para evidenciar cache hit instantáneo.
- Timeline de eventos (tecleo, debounce cumplido, cache hit/miss, render) para mostrar criterio de ingeniería.
- Explicar complejidad en UI: primer término O(n) sobre 5,000; repetidos O(1) por cache.
- Mostrar degradación controlada: skeleton al inicio + mensaje claro de error + recuperación sin recargar toda la app.
